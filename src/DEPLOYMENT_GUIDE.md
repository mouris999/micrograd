## üì± Platform-Specific Deployment

### üåê Web Apps

#### Output Files
When building for web, you'll receive:
- `index.html` - Main HTML file
- `styles.css` - Stylesheet (if generated)
- `script.js` - JavaScript code
- Additional assets (images, fonts, etc.)

#### Deployment Options

**Option 1: Vercel (Recommended)**
```bash
# Install Vercel CLI
npm install -g vercel

# Deploy
vercel

# Follow prompts to configure
```

**Option 2: Netlify**
1. Go to [Netlify](https://netlify.com)
2. Drag and drop your project folder
3. Site is live instantly

**Option 3: GitHub Pages**
```bash
# Create GitHub repo
git init
git add .
git commit -m "Initial commit"
git remote add origin <your-repo-url>
git push -u origin main

# Enable GitHub Pages in Settings
```

**Option 4: Traditional Hosting**
- Upload files via FTP to any web host
- Point domain to uploaded directory
- Ensure index.html is the root file

#### Prerequisites
- ‚úÖ All files in one directory
- ‚úÖ index.html as entry point
- ‚úÖ Relative paths for assets
- ‚úÖ CDN links for libraries

### ü§ñ Android Apps

#### Output Files
Android builds generate:
- `app.apk` - Installable package
- `app-release.aab` - Google Play bundle
- `AndroidManifest.xml` - App configuration
- `src/` - Source code
- `res/` - Resources (layouts, icons)

#### Local Testing

**Install APK on Device:**
```bash
# Enable USB debugging on Android device
# Connect device via USB

adb install app.apk
```

**Install on Emulator:**
```bash
# Start Android emulator
emulator -avd Pixel_5_API_30

# Install APK
adb install app.apk
```

#### Google Play Store Deployment

**Step 1: Prepare App**
1. Sign the APK
```bash
keytool -genkey -v -keystore my-release-key.keystore \
  -alias my-key-alias -keyalg RSA -keysize 2048 -validity 10000

jarsigner -verbose -sigalg SHA1withRSA -digestalg SHA1 \
  -keystore my-release-key.keystore app.apk my-key-alias
```

2. Optimize APK
```bash
zipalign -v 4 app.apk app-aligned.apk
```

**Step 2: Create Play Console Account**
- Visit [Google Play Console](https://play.google.com/console)
- Pay one-time $25 registration fee
- Complete account setup

**Step 3: Create App Listing**
1. Click "Create App"
2. Fill in app details:
   - App name
   - Default language
   - App/Game selection
   - Free/Paid

**Step 4: Upload Build**
1. Go to "Release" ‚Üí "Production"
2. Upload AAB file
3. Complete store listing:
   - App description
   - Screenshots
   - Feature graphic
   - Privacy policy URL

**Step 5: Content Rating**
- Complete content rating questionnaire
- Receive rating certificate

**Step 6: Pricing & Distribution**
- Set countries
- Choose free or paid
- Accept terms

**Step 7: Review & Publish**
- Review all sections
- Submit for review
- Wait 1-7 days for approval

#### Alternative: Direct Distribution

**Option 1: APK Direct Download**
- Host APK on your website
- Users download and install
- Requires "Install from Unknown Sources"

**Option 2: Third-Party Stores**
- Amazon Appstore
- Samsung Galaxy Store
- Aptoide

### üçé iOS Apps

#### Output Files
iOS builds generate:
- `App.ipa` - Installable package
- `ProjectName.xcodeproj` - Xcode project
- `Info.plist` - App configuration
- Swift/Objective-C source files
- Assets.xcassets - Images and icons

#### Local Testing

**Install on Simulator:**
```bash
# Open in Xcode
open ProjectName.xcodeproj

# Select simulator
# Click "Run" (Cmd+R)
```

**Install on Physical Device:**
1. Connect iPhone/iPad via USB
2. Trust computer on device
3. Select device in Xcode
4. Click "Run"

#### App Store Deployment

**Prerequisites:**
- Mac computer with Xcode
- Apple Developer Account ($99/year)
- Valid signing certificates

**Step 1: Apple Developer Account**
1. Visit [Apple Developer](https://developer.apple.com)
2. Enroll in program ($99/year)
3. Complete account setup

**Step 2: App ID & Certificates**
```bash
# In Xcode
1. Open project
2. Select target
3. Go to "Signing & Capabilities"
4. Select your team
5. Xcode auto-manages certificates
```

**Step 3: App Store Connect**
1. Go to [App Store Connect](https://appstoreconnect.apple.com)
2. Click "My Apps" ‚Üí "+" ‚Üí "New App"
3. Fill in:
   - Platform (iOS)
   - App name
   - Primary language
   - Bundle ID
   - SKU

**Step 4: Build & Archive**
```bash
# In Xcode
1. Select "Any iOS Device"
2. Product ‚Üí Archive
3. Wait for archive to complete
4. Click "Distribute App"
5. Choose "App Store Connect"
6. Upload to App Store
```

**Step 5: App Store Listing**
Complete in App Store Connect:
- App description
- Keywords
- Screenshots (all required sizes)
- App icon
- Privacy policy URL
- Support URL

**Step 6: App Review Information**
- Demo account (if needed)
- Notes for reviewer
- Contact information

**Step 7: Submit for Review**
1. Select build uploaded earlier
2. Complete all required fields
3. Submit for review
4. Wait 1-7 days for approval

#### TestFlight (Beta Testing)

**Distribute Before App Store:**
```bash
# After archiving in Xcode
1. Distribute App
2. Choose "TestFlight Internal Testing"
3. Add internal testers
4. They receive TestFlight invite
```

## ‚ö° Performance Optimization

### Web Apps

**Before Deployment:**
```bash
# Minify HTML
html-minifier index.html -o index.min.html

# Optimize images
imageoptim images/

# Bundle and minify JS
webpack --mode production

# Compress CSS
cssnano styles.css
```

**CDN Recommendations:**
- Cloudflare (free tier)
- AWS CloudFront
- Vercel Edge Network

### Android Apps

**Build Optimizations:**
```gradle
// In build.gradle
android {
    buildTypes {
        release {
            minifyEnabled true
            shrinkResources true
            proguardFiles getDefaultProguardFile('proguard-android.txt')
        }
    }
}
```

**Image Optimization:**
- Use WebP instead of PNG
- Compress images with TinyPNG
- Use vector drawables when possible

### iOS Apps

**Build Settings:**
```swift
// Optimization Level: Fastest, Smallest
// Strip Debug Symbols: Yes
// Enable Bitcode: Yes
```

**Image Optimization:**
- Use asset catalogs
- Provide @1x, @2x, @3x versions
- Use PDF vectors for icons

## üö® Common Issues

### Web Deployment

**Issue: CORS Errors**
```javascript
// Solution: Add CORS headers on server
Access-Control-Allow-Origin: *
```

**Issue: HTTPS Required**
```
Solution: Use Vercel/Netlify (auto HTTPS)
Or: Get free SSL from Let's Encrypt
```

### Android Deployment

**Issue: App Not Installing**
```
Solution: Check minimum SDK version
Ensure device meets requirements
```

**Issue: Play Store Rejection**
```
Common reasons:
- Missing privacy policy
- Incorrect content rating
- Crashes on testing
- Malware detected
```

### iOS Deployment

**Issue: Certificate Errors**
```
Solution: Regenerate certificates in Xcode
Ensure bundle ID matches
```

**Issue: App Store Rejection**
```
Common reasons:
- Crashes on launch
- Missing screenshots
- Privacy violations
- Guideline violations
```
